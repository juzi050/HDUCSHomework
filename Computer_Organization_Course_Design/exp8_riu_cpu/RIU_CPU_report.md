# 实验 8 RIU 多周期 CPU 实验报告

## 一、实验目的

本实验实现 RV32I 中 R 型运算类指令、I 型运算类指令和 U 型 `lui` 指令的数据通路与控制流程。通过二级译码、ALU、寄存器堆、暂存器、取指模块和有限状态机控制单元连接，构建一个可单步观察的多周期 RIU_CPU。

## 二、模块结构与控制流程

RIU_CPU 的主要数据通路为：

```text
PC -> IM -> IR -> ID1 -> ID2/CU
Regs(rs1/rs2) -> A/B latch -> ALU -> F latch -> Regs(rd)
imm32 --------------------------^
```

控制单元状态机如下：

| 状态 | 编码 | 动作 | 下一状态 |
| --- | --- | --- | --- |
| `S1` | `001` | `PC_Write=1`、`IR_Write=1`，取指并令 PC 加 4 | `S2` |
| `S2` | `010` | `AB_Write=1`，读 `rs1/rs2` 到 A/B | R 型到 `S3`，I 型到 `S5`，`lui` 到 `S6` |
| `S3` | `011` | R 型 ALU 运算，结果写入 F | `S4` |
| `S5` | `101` | I 型 ALU 运算，B 端选择 `imm32`，结果写入 F | `S4` |
| `S6` | `110` | `lui` 运算，F 写入 `imm32` | `S4` |
| `S4` | `100` | `Reg_Write=1`，F 写回 `rd` | `S1` |

二级译码模块 `ID2` 根据 `opcode/funct3/funct7` 生成指令类型和 ALU 操作码。R 型、I 型和 `lui` 均按 4 个 CPU 单步时钟完成。

## 四、仿真方案与结论

仿真测试包含 4 个自检 testbench：

| testbench | 验证内容 |
| --- | --- |
| `alu_tb` | 加减、逻辑、移位、有符号/无符号比较、`pass_b/lui` 和标志位 |
| `id2_tb` | 10 条 R 型、9 条 I 型、`lui` 和非法指令的二级译码 |
| `riu_cpu_tb` | 完整运行 27 条测试程序，逐条检查 `IR/ST/W_Data/寄存器写回` |
| `top_tb` | 单步按键、复位、数码管显示选择、LED 与交通灯输出 |

本次使用 Vivado 2025.1 执行 `scripts/run_all.tcl`，上述 4 个 testbench 均输出 `ALL TESTS PASSED`。综合实现后生成 `build/RIU_CPU.bit`，时序报告显示 `WNS=0.403ns`、`TNS=0.000ns`，满足 100MHz 时钟约束。

结论：

1. I 型运算指令执行正确：`riu_cpu_tb` 检查了 `addi/slli/slti/sltiu/xori/srli/srai/ori/andi` 的状态流 `S1->S2->S5->S4` 和写回结果。
2. R 型运算指令执行正确：`add/sub/sll/srl/sra/slt/sltu/and/or/xor` 均按 `S1->S2->S3->S4` 完成，并写入预期寄存器。
3. `lui` 指令执行正确：`lui x13,0x80000` 和 `lui x24,0x00010` 均按 `S1->S2->S6->S4` 完成，写回值分别为 `0x8000_0000` 和 `0x0001_0000`。

## 五、板级验证方案

板级顶层 `top` 使用 HCS-A02 的 100MHz 时钟作为系统时钟，`BT0` 产生单周期 `step_en`，每按一次推进一个 CPU 状态；`BT1` 为复位按钮，按下后 PC、IR、寄存器堆和暂存器清零，CU 回到 `S1`。

操作过程：

1. 按下 `BT1` 复位后松开，确认 `SW[2:0]=000` 时数码管显示 `PC=00000000`。
2. 每按一次 `BT0`，观察 `ST` 是否按 4 状态流程变化。
3. 对每条指令，在 `S4` 写回阶段观察 `W_Data`，与上表对照。
4. 使用 `SW[2:0]` 切换显示 `PC/IR/W_Data/ST/flags`。
5. `LD[34:32]` 直接显示当前 `ST`，`LD[31:28]` 显示 `{ZF,CF,OF,SF}`，`LD[27:0]` 显示 `W_Data` 低 28 位。

## 六、思考与探索

1. 二级译码的作用：ID1 只按固定字段拆分指令，ID2 再结合 `opcode/funct3/funct7` 判断真实操作，能把字段解析和控制生成解耦。
2. 多周期 CPU 的特点：一条指令拆成取指、译码读寄存器、执行、写回多个状态，同一套 ALU 和寄存器堆可在不同周期复用。
3. 同步 ROM 的取指：ROM 输出随系统时钟更新，PC 在 `S1` 自增后，下一条指令会在后续状态中提前稳定到 ROM 输出端，下一次 `S1` 可锁存到 IR。
4. 移位立即数编码：`slli/srli/srai` 都属于 I 型，但移位量只使用 `imm[4:0]`；`srai` 通过 `funct7=0100000` 与 `srli` 区分。
5. 有符号与无符号比较差异：`slt/slti` 使用补码有符号比较，`sltu/sltiu` 使用无符号比较。例如 `0xF000_0024` 有符号为负数，和无符号数比较时结论可能不同。
