## 思考题 1

在 CPU 上直接使用四重循环做卷积，瓶颈主要来自 Python 层循环次数过多，导致大量时间消耗在解释器调度而不是数值计算上。更高效的做法是先把输入特征图按卷积窗口展开成二维矩阵，也就是常说的 `im2col`。展开后，每一行对应一个局部感受野。

卷积核也可以重排成二维矩阵，形状从 `Cin x K x K x Cout` 变成 `(Cin*K*K) x Cout`。这样原来“每个窗口和每个卷积核做一次内积”的过程，就能统一写成一次矩阵乘法，再加偏置即可得到整层输出。这个思路在本实验代码里对应 `forward_speedup` 中的 `img2col` 与 `weight_reshape`。

这种实现的加速来源主要有两点：一是显著减少 Python 显式循环；二是把核心计算交给底层高效线性代数库完成。代价是需要额外的中间展开矩阵，因此会增加内存占用，分辨率越大、通道越多，这个代价越明显。

## 运行说明

- 代码使用的 VGG19 权重文件为 `imagenet-vgg-verydeep-19.mat`，本地运行时放在 `风格迁移/` 目录下即可。
- 由于当前环境中的 `scipy.misc` 已不可用，仓库中额外保留了预处理后的 `weinisi.npy` 和 `style.npy` 作为输入资源。
- 本次实现保留了卷积和池化层的加速路径，并把多层损失合并为一次反向传播，减少 CPU 训练时间。
