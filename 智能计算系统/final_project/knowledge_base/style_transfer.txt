【风格迁移概述】
基于 VGG19 的图像风格迁移：将参考风格图的风格应用到内容图上，生成一幅同时保持内容图内容和风格图风格的图像。
核心思想：利用预训练的 VGG19 网络提取不同层的特征，用特征之间距离定义内容损失和风格损失。

【VGG19 网络架构】
VGG19 共包含 37 层（按顺序）：
  conv1_1, relu1_1, conv1_2, relu1_2, pool1,
  conv2_1, relu2_1, conv2_2, relu2_2, pool2,
  conv3_1, relu3_1, conv3_2, relu3_2, conv3_3, relu3_3, conv3_4, relu3_4, pool3,
  conv4_1, relu4_1, conv4_2, relu4_2, conv4_3, relu4_3, conv4_4, relu4_4, pool4,
  conv5_1, relu5_1, conv5_2, relu5_2, conv5_3, relu5_3, conv5_4, relu5_4, pool5

分为 5 个 block，每个 block 包含若干卷积层和 ReLU 层，最后接最大池化层。
Block 1: 2 个卷积层 (channels: 3->64->64)
Block 2: 2 个卷积层 (channels: 64->128->128)
Block 3: 4 个卷积层 (channels: 128->256->256->256->256)
Block 4: 4 个卷积层 (channels: 256->512->512->512->512)
Block 5: 4 个卷积层 (channels: 512->512->512->512->512)
卷积核大小均为 3x3，步长为 1，填充为 1（保持特征图大小不变）。
池化窗口为 2x2，步长为 2（特征图大小减半）。

【内容损失（Content Loss）】
内容损失衡量生成图像和内容图像在 VGG19 某一层的特征图之间的差异。
公式：L_content = 0.5 * Σ(F_transfer - F_content)²
其中 F 是某一层的输出特征图。
实现为均方误差（MSE）：0.5 * ||F_transfer - F_content||²

内容损失的层选择（实验中使用）：
- 内容损失层：['relu4_2']
- 原因：relu4_2 位于 VGG19 的较深层，该层的特征图已经保留了高级语义信息（物体的形状和结构），
  但同时具有足够的分辨率来保持空间细节。较浅层（如 relu1/2）太底层（保留太多纹理），
  较深层（如 relu5）太抽象（丢失空间结构）。

【风格损失（Style Loss）】
风格损失使用 Gram 矩阵来衡量生成图像和风格图像的特征相关性。
Gram 矩阵公式：G = F * F^T
其中 F 被 reshape 为 [C, H*W]，G 的形状为 [C, C]。
Gram 矩阵捕获了特征图各通道之间的相关性，代表了图像的"风格"（纹理、颜色分布等）。

风格损失公式：
  L_style = Σ(G_transfer - G_style)² / (4 * C² * (H*W)²)
其中 C 是通道数，H*W 是特征图的空间大小。

风格损失的层选择（实验中使用）：
- 风格损失层：['relu1_1', 'relu2_1', 'relu3_1', 'relu4_1', 'relu5_1']
- 原因：风格信息分布在不同深度层上。浅层捕获颜色和简单纹理，深层捕获更复杂的模式。
  使用多个层的组合可以更好地捕捉多尺度的风格特征。每个 block 的第一层被选为代表。

【总损失函数】
L_total = α * L_content + β * L_style
其中 α 和 β 是权重系数。

【超参数选择】
课程实验中的超参数配置：
- ALPHA (α) = 1.0 — 内容损失权重
- BETA (β) = 500.0 — 风格损失权重
- NOISE = 0.5 — 初始噪声与内容图混合比例
- LEARNING_RATE = 1.0 — 学习率
- IMAGE_HEIGHT = 192 — 图像高度
- IMAGE_WIDTH = 320 — 图像宽度
- 训练总步数 TRAIN_STEP = 201
- 最佳步数 best_step = 200（best_loss = 1.55e11）
- 优化器：Adam（beta1=0.9, beta2=0.999, epsilon=1e-8）

α 和 β 的选择原理：
- α 控制内容保留程度：α 越大，生成图像越接近内容图
- β 控制风格迁移强度：β 越大，风格化越强烈
- β/α 比值决定风格化程度：比值过大则内容被破坏，比值过小则风格不明显
- 实验发现 β=500, α=1 时取得了较好平衡

【im2col 卷积加速】
在 CPU 上使用四重循环做卷积的主要瓶颈是 Python 层循环次数过多，导致大量时间消耗在解释器调度上。
更高效的做法是 im2col：
1. 先将输入特征图按卷积窗口展开成二维矩阵（每一行对应一个局部感受野）
2. 将卷积核重排成二维矩阵，形状从 [C_in, K, K, C_out] 变为 [C_in*K*K, C_out]
3. 用矩阵乘法完成"每个窗口和每个卷积核做内积"的过程
4. 加速来源：减少 Python 显式循环 + 底层 BLAS 高效矩阵乘法
5. 代价：额外的中间展开矩阵占用内存

【Adam 优化器】
课程中从头实现了 Adam 优化器：
  m_t = beta1 * m_{t-1} + (1 - beta1) * grad          — 一阶矩估计（动量）
  v_t = beta2 * v_{t-1} + (1 - beta2) * grad²         — 二阶矩估计（梯度平方）
  m_hat = m_t / (1 - beta1^t)                         — 偏差校正
  v_hat = v_t / (1 - beta2^t)                         — 偏差校正
  output = input - lr * m_hat / (sqrt(v_hat) + eps)   — 参数更新

【最大池化（Max Pooling）】
池化窗口 k×k，步长 s（通常 k=2, s=2）：
- 在池化窗口内取最大值作为输出
- 前向传播：输出 = max(窗口内值)
- 反向传播：梯度仅传给最大值位置，其余位置梯度为 0
- 需要存储 argmax（最大值位置）用于反向传播
